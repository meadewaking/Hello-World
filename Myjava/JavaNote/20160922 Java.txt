1. 异常处理机制
异常处理机制用来处理，程序运行过程中发生的不至于使应用程序崩溃的错误，异常处理机制使得应用程序的健壮性更强。
（1）语法：
try {
	// 可能发生异常的语句
	// 执行try块中的语句时，如果发生异常则该语句和其后的语句将不再执行，同时产生异常对象并抛出
} catch (  ) {	// catch块将根据异常类型捕获异常对象并处理
	// 异常处理时通常只做提示处理不做挽救处理
} catch (  ) {	// catch块可以有多个，先处理具体类型的异常
} 
（2）适用范围
a. 输入数值时
b. 有除法且除数是变量
c. 文件操作时
d. 数据库访问时
e. 其它情况
2. 自定义异常
（1）创建异常类：
a. 命名，并以Exception为后缀；
b. 自定义异常类必须是Exception类的派生类；
c. 通常需要定义无参构造方法，并在构造该类对象时明确异常原因描述；
（2）自定义异常的使用：
a. 确定异常触发位置和触发条件；
b. 在符合触发条件时创建异常对象并抛出；
